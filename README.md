## 🎯 О проекте

Решение для соревнования по машинному обучению **DGA Domain Detection Challenge**, направленное на обнаружение алгоритмически сгенерированных доменных имен (DGA), используемых вредоносным ПО для связи с командными серверами.

**Ключевая задача:** Минимизировать ложные срабатывания (false positives), так как блокировка легитимного домена критичнее, чем пропуск одного DGA-домена.

### 📊 Масштаб данных
- **Тренировочный набор:** 17,719,790 доменов
- **Тестовый набор:** 7,594,197 доменов
- **Признаки:** Только доменные имена (текстовые строки)

## 🦠 Проблема DGA

### Что такое DGA?
**Domain Generation Algorithm (DGA)** — это алгоритмы автоматической генерации большого количества доменных имен, используемые вредоносным ПО для:
- 🕵️ Обхода систем блокировки
- 🤖 Связи с командными серверами (C&C)
- 🌐 Управления ботнетами
- Статистика угрозы:

🔄 Ежедневно генерируются тысячи новых DGA доменов

⚡ Традиционные черные списки неэффективны

🎯 Требуется поведенческий анализ в реальном времени

## Почему F₀.₅ метрика?
F₀.₅ = (1 + 0.5²) × (Precision × Recall) / (0.5² × Precision + Recall)

# β = 0.5 означает: Precision в 2 раза важнее Recall
False Positive (легитимный → DGA)	Блокировка настоящего сайта, нарушение работы пользователей	🔴 Критично
False Negative (DGA → легитимный)	Пропуск одного из тысяч DGA доменов	🟡 Допустимо
